(N)Vim - How I Came To Love It

Cerita ini awalnya dimulai pada tahun 2018 pada saat tahun pertama perkuliahan. Di salah satu jajaran mata kuliah yang saya ambil, terdapat "DKP" (Dasar Komputer dan Pemrograman), dimana di MK tersebut kita diajarkan mengenai dasar pemrograman dan kiat-kiat pemrograman yang terbaik. Sebenarnya saya sendiri tidak masalah dengan pembelajarannya, karena di waktu liburan, saya sudah mulai mengoprek Arduino saya, dan kebetulan bahasa pemrograman yang digunakan di MK DKP juga sama-sama C. Namun, saya mengalami kebingungan dengan workflow yang digunakan di MK tersebut. Jadi ceritanya, untuk melakukan sentralisasi data, dosen pengajar mengharuskan kita untuk mengupload / mengerjakan program-program tugas dengan login SSH ke server yang beliau sediakan. Coming from non-IT background, obviously, I was shocked, because not only logging in to the server using SSH does not give us interactive experience, it does not have GUI! WE WERE ONLY GIVEN A PLAIN TERMINAL SESSION WITH BASH SHELL. I WAS REALLY SHOCKED. Lanjut, di server tersebut kita diberi pilihan "text editor" yang bebas digunakan, beberapa diantaranya adalah "nano", "vi", dan "emacs" (I know emacs is not a text editor, but bare with me, I know nothing at that time). Dan saya coba-coba saja semuanya, yah, seperti dugaan tidak ada yang nyaman. Karena saya memang pengalaman selama ini hanya menggunakan text editor yang pokoknya bisa klik-klik pakai mouse, bukan yang keyboard-based. Yahhhh, seminggu 2 minggu, saya mulai terbiasa, dan waktu itu saya putuskan untuk stay menggunakan "vi", namun tetap saja istilahnya "gratul gratul" (istilah Jawa untuk melakukan sesuatu secara tidak lancar). Nah, karena saya tidak memiliki background pengetahuan IT sebelumnya, saya jadi sering bertanya-tanya ke om saya pada waktu itu, karena memang om saya bekerja di bidang IT (waktu itu saya taunya programmer Python). Dan om saya menyarankan, "mungkin yeye bisa coba-coba text editor lain, atau kalau coba pakai Linux juga bisa, nanti kerja di Linux nya yeye, terus file nya bisa di copy ke server dosen nya pakai SSH". Berkenalanlah saya dengan "Virtual Machine", dimana saya waktu itu memilih untuk mengambil Ubuntu sebagai Linux pertama saya, dan saya rasa kebanyakan orang juga memilih distro yang berbasis Ubuntu sebagai distro pertamanya. Singkat cerita, saya mulai coba-coba berbagai macam text editor, dan akhirnya stay cukup lama di Sublime Text & Visual Studio Code. Lalu pada suatu hari, saya kebetulan melihat om saya sedang bekerja di laptop Macbook-nya, dan saya langsung mikir "hah, ini text editor apaan bisa gerak sana sini tanpa mouse, cepat gerakannya pula, dan fitur-fiturnya juga ga kalah sama Visual Studio Code" (waktu itu saya cari text editor pokoknya ada auto completion nya). Saya coba tanya ke om saya saja, kok keliatannya seru ini text editor. Om saya jawab "loh ini Vi(m) ye, kan dulu yeye sudah pernah pakai Vi, nah Vim itu semacam pengembangan lebih lanjutnya". Aku be like: "HAH, ITU TEXT EDITOR DARI VI??? GA PAKAI MOUSE BISA KAYAK GITU???". Habis itu saya mulai googling panjang lebar soal Vim, saya bahkan sampai membuat cheatsheet catatan kecil-kecilan untuk lebih mudah memahami "verb" dan "text object" yang ada di "language of vim". Dan makin kesini, saya makin paham, bahwa sebenarnya Vim ini text editor yang kalau dipakai itu tidak perlu mikir. Katakanlah saya ingin menghapus 2 kata, saya bisa menekan (di NORMAL mode) "2diw". "d" artinya "delete", "i" artinya "inner" ini agar dimanapun kursor berada di kata tersebut semua huruf dalam kata tersebut akan terhapus, "w" artinya "word". Mau mengubah 2 kata? Gampang! tinggal ganti kata kerja (verb) "d" menjadi "c" yang di Vim artinya "change" (-> "change inner word"). Dan masih banyak "verb", "motion", "text object" lain, yang bahkan saya sekarang tidak bisa sebutkan satu persatu, tapi saya cukup memikirkan "saya mau ngapain di text ini", nanti otomatis pasti keluar sendiri "apa yang harus ditekan". Karena menurut saya keymap yang harus ditekan di Vim itu "logical" (in a way mirip bahasa yang bisa dikomunikasikan). Sekarang, saya sudah hampir 4 tahun menggunakan Vim, dan saya sudah memiliki setup Neovim yang bisa dibilang "powerful" dan "lightweight". Berbeda dengan VSCode yang harus menunggu 5-6 detik untuk launching, Neovim bahkan memiliki startup time dalam orde milli-second. Dan selama perjalanan itu juga, saya jadi belajar banyak soal dunia Linux dan Vim/Neovim.
